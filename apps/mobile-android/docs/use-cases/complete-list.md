# Caso de uso: Completar lista

## Objetivo
Permitir al usuario finalizar una lista aunque queden productos sin comprar.

## Precondiciones
- Usuario autenticado.
- Lista activa abierta.

## Flujo principal
1) El usuario toca el botón "Completar lista".
2) Se muestra confirmación.
3) Si confirma, se marca la lista como completada.

## Reglas y decisiones
- La lista puede completarse aunque haya productos sin comprar.
- La confirmación es obligatoria.

## Mensajes sugeridos
- Confirmación: "¿Completar la lista? Puedes finalizar aunque queden productos."
